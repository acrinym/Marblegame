# Pathing System & Connection Logic (The "Spine" of the Machine)

## 1. The Core Misconception
**CURRENT FAILURE:** The AI treats components as isolated islands.
**REQUIRED LOGIC:** The game is a **Continuous Flow System**. A "Path" is not just a visual line; it is a strict **Parent-Child dependency**.

**Rule #1:** A marble must *never* touch "empty air" between components.
**Rule #2:** If Component A connects to Component B, Component B's position is **calculated relative** to Component A's *Exit Port*, not placed arbitrarily.

---

## 2. Anatomy of a Component
Every contraption (Funnel, Track, Diverter) must expose standard "Ports" in its code interface.

### The `Component` Interface
```typescript
interface Component {
  id: string;
  type: string;
  x: number;  // Center X
  y: number;  // Center Y
  
  // The vital connection points (Relative to center X,Y)
  inputs: { xOffset: number, yOffset: number }[]; 
  exits:  { xOffset: number, yOffset: number }[];
}

```

### Example: The "Standard Curve" (Quarter Pipe)

* **Input:** Top Center (0, -radius)
* **Exit:** Bottom Right (+radius, 0)
* **Connection Logic:** If we attach this to a previous piece, we align this `Input` with the previous piece's `Exit`.

---

## 3. How "Paths" Work (The Grid System)

We cannot use free-form coordinates for a puzzle game. We must use a **Node-Link System** or a **Snapping Grid**.

### A. The "Snapping" Algorithm

When building a level, we don't say "Place Track at 500, 500." We say:

> "Attach a **Right-Curve** to the **Bottom-Exit** of **Funnel_1**."

**The Math:**

1. Get `Funnel_1` world position: `(Fx, Fy)`.
2. Get `Funnel_1` Exit Offset: `(0, +50)`.
3. **Anchor Point:** `(Fx, Fy + 50)`.
4. Get `Right-Curve` Input Offset: `(0, -50)` (assuming top entry).
5. **New Component Position:** `(AnchorX - InputOffset.x, AnchorY - InputOffset.y)`.

### B. Visualizing the "Path"

The "Path" is the **negative space** inside the rails.

* **Wrong:** Drawing two lines and hoping the marble fits.
* **Right:** Define the *centerline* curve first. Then generate the left rail at `center - width/2` and right rail at `center + width/2`.

---

## 4. The "Connector" Pieces (The Glue)

The AI keeps forgetting that straight lines don't just "happen." We need explicit connector types to bridge gaps.

| Piece | Function | Visual |
| --- | --- | --- |
| **Straight Rail** | Connects two aligned ports. | Two parallel rectangles. |
| **Flex-Hose** | Connects misaligned ports. | A Bezier curve generated rail. |
| **Drop Tube** | A vertical fall. | Two vertical walls (Gravity does the work). |
